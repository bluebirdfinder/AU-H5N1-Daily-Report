# 澳洲 H5N1 疫情與 Nestlé Blayney 廠地緣風險自動化報告系統

本專案為獨立之澳洲 H5N1 禽流感疫情 GIS 自動更新報告系統。透過 GitHub Actions 每天自動爬取澳洲與地方政府官網及新聞 RSS 最新數據，結合地理編碼 API 自動更新網頁 GIS 地圖，並自動部署發布於 GitHub Pages 上。

---

## 🌟 核心功能特點
1. **全澳州聯防監控網絡與防改版 RSS 兜底**：同時爬取**澳洲聯邦農業部 (DAFF)** 以及**澳洲全部 8 個州/領地政府**的官方禽流感站點。若任何官方網址因改版失效，程式會啟動兜底防線直接從 Google News RSS 提取新疫情地點，100% 確保數據不漏報。
2. **AI 智慧地理定位與多段退避 (Nominatim API)**：當新聞或官網出現全新疫情地點時，程式會自動使用 OpenStreetMap 地理編碼 API 查詢其精確 GPS 經緯度，並實作多段退避機制（自動剔除海灣沙灘等字尾）防止定位失敗，自動在網頁地圖上標示新病例。
3. **媒體確診交叉驗證 (C 方案)**：結合「主流媒體白名單」與「官方首長/國家實驗室發言人及確診詞」雙重過濾，自動將媒體報導的最新疫情於官方網站同步延遲期間內搶先升級，並於前端地圖與表格自動標示精緻的 **`⚠️ 媒體先行 (官網同步中)`** Badge，保證極高的時效性與數據真實性。
4. **動態最短距離計算 (Haversine 公式)**：自動計算各病例到 Blayney 廠的球體直線距離，並在網頁頂部標籤、地緣安全宣告、最下方三大黃金論點第 3 點等三處，自動更新為最新的最近距離。
5. **自適應動態參考文獻庫**：底部的參考資料（References）完全動態生成，會根據當前數據庫中病例所分布的省份（如西澳 WA、維多利亞州 VIC），自動追加該省政府農業廳的官方監控網址。
6. **雙時區對齊**：最後編譯時間自動校正顯示台北時間與澳洲 AEST 時間。
7. **陰性排除案例自動判定與更新**：升級爬蟲判定邏輯與通用更新機制，能自動識別官方報告與即時新聞中的陰性排除案例（如雪梨 Narrabeen Beach），實現狀態的自動雙向更新，最大程度提供科學數據的安心保障。

## 📂 檔案目錄結構
* **`h5n1.py`**：自動爬取官方與新聞 RSS，自動定位新地點並編譯輸出 `index.html` 的 Python 核心引擎。
* **`report_template.html`**：網頁 GIS 報告模板（整合 Leaflet.js 地圖與 Tailwind CSS 樣式，供 `h5n1.py` 讀取）。
* **`.github/workflows/auto_update.yml`**：GitHub Actions 定時自動化工作流設定檔（每天定時執行 2 次）。
* **`index.html`**：編譯後生成的正式報告網頁。
* **`更新說明`**：最後檔案同步日期更新於 2026-07-22 15:00。
